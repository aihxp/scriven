| H-02 | HIGH   | `copyDirWithPreservation` routes the final write through `atomicWriteFileSync` — no truncation window on mid-copy crash |
| M-01 | MEDIUM | `copyDirWithPreservation` gates on `lstatSync`; non-regular dest entries (symlinks etc.) are backed up then replaced with shipped content |
| M-02 | MEDIUM | `cleanOrphanedTempFiles` recurses with a depth cap of 4 so nested orphan tmp files under `skills/<name>/SKILL.md.tmp.<uuid>` are swept |
| M-03 | MEDIUM | `readFrontmatterValues` detects YAML block-scalar indicators (`|`, `>`) and warns + falls back to empty string |
| M-04 | MEDIUM | `writeSharedAssets` runs `migrateSettings` + `validateSettings` on the existing file; invalid files are preserved as `settings.json.invalid.<timestamp>` |
| L-01 | LOW    | Orphan-tmp UUID regex tightened to the canonical 8-4-4-4-12 shape (`[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}`) |
| L-02 | LOW    | `readFrontmatterValues` warns on duplicate keys; first-occurrence-wins semantics retained |
