The command file must include:

**Frontmatter:**
- `description: Run the full drafting pipeline autonomously. Choose guided, supervised, or full-auto profiles.`
- `argument-hint: "[--profile guided|supervised|full-auto] [--from <stage>] [--to <stage>] [--unit N] [--resume]"`

**Body sections:**

1. **"# Autopilot"** heading with role description: "You are running the Scriven pipeline autonomously. Your job is to execute the discuss-plan-draft-review-submit chain for each unit in OUTLINE.md, pausing only when the active profile requires it."

2. **"## What to do"** section with numbered steps:
   - Read config.json for autopilot settings (profile, enabled, custom_checkpoints)
   - Read STATE.md for current position (current unit, current stage, progress)
   - Read OUTLINE.md for total units and order
   - If `--resume` flag: read STATE.md for last completed unit and current stage, output ONE sentence explaining where it left off (per D-03), then continue the loop from that point
   - Enter the main loop (same routing logic as /scr:next but in a loop)

3. **"## Main loop"** section with the loop pseudocode:
   ```
   FOR each unit in OUTLINE.md (starting from current position):
     FOR each stage in [discuss, plan, draft, review, submit]:
       Execute the stage command (/scr:discuss-{unit}, /scr:plan-{unit}, etc.)
       Update STATE.md with: timestamp, command, unit, outcome
       Check pause conditions (see profile rules below)
       If paused: show review prompt, wait for input
       If "stop": exit loop, save state
       If "skip": advance to next unit
   ```

4. **"## Profile rules"** section defining all three profiles:

   **Guided (per D-01):**
   - Pause after EVERY atomic unit draft completion
   - At each pause, show: (a) one-sentence summary of what was drafted, (b) the last 200 words of the drafted prose, (c) prompt "approve / revise / stop"
   - On "approve": continue to next unit
   - On "revise": re-enter draft stage for that unit with writer's feedback
   - On "stop": save state and exit

   **Supervised (per D-04):**
   - Batch by structural unit boundary: use the `mid` level from the work type hierarchy in CONSTRAINTS.json
   - For a novel: hierarchy.mid = "chapter", so pause when all scenes in a chapter are drafted
   - For a screenplay: hierarchy.mid = "sequence", pause when all scenes in a sequence are drafted
   - At each pause, show: list of completed units with word counts, prompt "continue / review [unit] / stop"
   - To detect boundary: read OUTLINE.md to know which atomic units belong to which mid-level unit; pause when all atomic units in that mid-level unit are complete

   **Full-auto (per D-02):**
   - Run until ALL units are complete
   - Pause ONLY on these conditions:
     (a) Continuity contradiction the agent cannot resolve
     (b) Voice drift exceeding config.voice.drift_threshold (default 0.3)
     (c) Plot hole with no clear resolution
     (d) Missing critical information (character motivation gap, setting inconsistency)
     (e) Writer-defined checkpoints from config.autopilot.custom_checkpoints array
   - For writer-defined checkpoints: read the checkpoint text as natural-language instruction, check against OUTLINE.md structure at each iteration (e.g., "pause after each act climax" -- check if just-completed unit is marked as climax)
   - On pause: show diagnostic report (what triggered the pause, context, options) and wait
   - On completion: show comprehensive summary (total units, word count, voice consistency, any flags)

5. **"## Resume logic"** section (per D-03):
   - Read STATE.md "Session handoff" and "Progress" sections
   - Determine last completed unit and last stage
   - Output: "Last time, you [action from STATE.md]. Picking up at [unit] [stage]."
   - Re-enter main loop from that point

6. **"## Adaptive naming"** section:
   - Same pattern as next.md: load config.json for command_unit, use adapted names
   - Reference CONSTRAINTS.json for work-type-specific command names

7. **"## State management"** section:
   - MANDATORY: Update STATE.md after EVERY action (not just pauses)
   - Record in "Last actions" table: timestamp, command name, unit, outcome
   - Update progress counters (units_discussed, units_planned, etc.)
   - On pause: write current position so --resume can pick up

8. **"## Tone"** section:
   - Progress updates: concise, one line per unit ("Drafted chapter 3: 1,247 words. Voice check: passed.")
   - At pause points: warm, non-technical ("Chapter 3 is ready for your review.")
   - Never show: git terminology, file paths, technical jargon (unless developer_mode is true)

9. **"## Anti-patterns"** section:
   - NEVER run without updating STATE.md -- resume depends on accurate state
   - NEVER show raw git output in writer mode
   - NEVER skip the voice comparison after drafting (even if voice-check command from Phase 4 is not yet available -- use the drafter's built-in self-check)
   - NEVER re-draft a unit that STATE.md shows as already submitted
