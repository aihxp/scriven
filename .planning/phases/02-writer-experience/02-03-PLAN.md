**Create `commands/scr/pause-work.md`** (per D-09, AUTO-11):

Frontmatter:
- `description: Pause your work session. Captures where you are and what you were thinking so you can pick up later.`
- `argument-hint: ""`

Body sections:

1. **"# Pause Work"** -- "You are helping the writer pause their session gracefully. Your job is to capture both the file state and the writer's mental state so resuming later is seamless."

2. **"## What to do"**:
   - Read STATE.md for current position (unit, stage, progress)
   - Ask the writer (per D-09): "Any notes for when you come back?" -- This captures their thinking, intentions, concerns.
   - If writer provides notes: store in STATE.md "Session handoff" > "Resume context" along with the automated context
   - If writer says nothing: generate context automatically from STATE.md
   - Update STATE.md "Session handoff" section:
     ```
     **Last session ended:** {current timestamp}
     **Resume context:** {Automated: "Finished drafting chapter 3 (4 scenes, 1,247 words). Voice check passed. Was about to start discussing chapter 4."} {Writer's note: "I want chapter 4 to be shorter and more tense -- Marcus discovers the letter here."}
     ```
   - Auto-save if there are uncommitted changes (run `/scr:save` logic with message "Saved before pausing")
   - Tell the writer: "Paused. When you're ready to come back, just run `/scr:resume-work`."

3. **"## State capture"**:
   - Current unit and stage
   - Last few actions from "Last actions" table
   - Word count progress
   - Any pending flags (voice issues, continuity flags, open revisions)
   - Writer's mental notes (the most important part -- D-09 specifically asks for this)

4. **"## Tone"**: Warm, unhurried. "Take your time. Everything is saved. When you come back, I'll know exactly where we were."

**Create `commands/scr/resume-work.md`** (per D-10, AUTO-12):

Frontmatter:
- `description: Pick up where you left off. Reads your last session and tells you what's next.`
- `argument-hint: ""`

Body sections:

1. **"# Resume Work"** -- "You are welcoming the writer back and orienting them. Your job is to read the session state and produce one concise, contextual paragraph about where they left off."

2. **"## What to do"**:
   - Read STATE.md "Session handoff" section for: last session end time, resume context, writer's notes
   - Read STATE.md "Progress" section for: overall progress metrics
   - Read STATE.md "Pending" section for: next step, open revisions, unresolved notes
   - Generate ONE paragraph (per D-10):
     ```
     Last time you drafted chapter 3 (1,247 words across 4 scenes, voice check passed). You were working on chapter 4 -- you noted you wanted it shorter and more tense, with Marcus discovering the letter. I'd suggest starting with /scr:discuss-chapter 4 to shape the plan.
     ```
   - The paragraph has three parts: (a) what was done, (b) what was in progress + writer's notes, (c) suggestion for next step

3. **"## Output format"** (per D-10):
   - ONE paragraph, not a bulleted list or table
   - Include the writer's own notes if they left any (this is what makes it feel personal)
   - Suggest the logical next command
   - End with: "Ready to continue? Run `/scr:next` or tell me what you'd like to do."

4. **"## Edge cases"**:
   - No previous session: "This looks like a fresh start. Run `/scr:next` to get going."
   - Session handoff section empty: Generate context from Progress and Last actions tables
   - Very old session (>7 days): "It's been a while since your last session ([date]). Here's where things stand: ..."

**Create `commands/scr/session-report.md`** (per D-11, AUTO-13):

Frontmatter:
- `description: See what you accomplished this session. Shows units drafted, words written, and time spent.`
- `argument-hint: ""`

Body sections:

1. **"# Session Report"** -- "You are summarizing the writer's current session. Your job is to compute actionable metrics from STATE.md and present them clearly."

2. **"## What to do"**:
   - Read STATE.md "Last actions" table
   - Identify session boundaries: session started at the first action after the last `/scr:pause-work` or `/scr:resume-work` entry (or the first action if no pause/resume exists)
   - Compute metrics (per D-11):
     - **Units drafted:** Count distinct units that went through "draft" stage this session
     - **Words written:** Sum word counts from draft actions this session (from STATE.md entries or by counting words in newly created DRAFT.md files)
     - **Time estimate:** Difference between first and last action timestamps this session
     - **Quality passes run:** Count voice-check, continuity-check, editor-review actions this session

3. **"## Output format"** (per D-11):
   ```
   ## Session Report

   **Duration:** ~2 hours 15 minutes
   **Units drafted:** 1 chapter (4 scenes)
   **Words written:** 1,247
   **Quality passes:** voice-check (passed)

   **Actions this session:**
   | Time | What happened |
   |------|---------------|
   | 1:00 PM | Started session |
   | 1:15 PM | Discussed chapter 3 |
   | 1:30 PM | Planned chapter 3 (4 scenes) |
   | 2:45 PM | Drafted chapter 3 (1,247 words) |
   | 3:00 PM | Voice check passed |
   | 3:15 PM | Editor review complete |
   ```

4. **"## Edge cases"**:
   - No actions this session: "Nothing to report yet. Start working with `/scr:next`."
   - Missing timestamps: Estimate from git log if available
   - Per D-12: session state is per-project. Do not reference other projects.
