**Create `commands/scr/save.md`** (per D-05, AUTO-06):

Frontmatter:
- `description: Save your current work. Auto-generates a descriptive save message from context.`
- `argument-hint: "[optional message]"`

Body sections:

1. **"# Save"** -- "You are saving the writer's current work. Your job is to create a git commit with a writer-friendly message, without exposing any git terminology."

2. **"## What to do"**:
   - Check for `.git/` directory. If missing, run `git init` silently (in writer mode) or with explanation (in developer mode)
   - Read STATE.md for: current stage, current unit name/number, last command run
   - Auto-generate commit message from context (per D-05):
     - After drafting: "Saved after drafting {unit_name} {N}"
     - After review: "Saved after editor review of {unit_name} {N}"
     - After revision: "Saved after revising {unit_name} {N}"
     - Manual save: "Saved work in progress on {unit_name} {N}"
     - If writer provides optional message: "Saved: {writer's message}"
   - Run: `git add .manuscript/ && git commit -m "{generated message}"`
   - Update STATE.md "Last actions" table with timestamp, "save", unit, message
   - Tell the writer: "Saved." (writer mode) or "Committed: {hash} - {message}" (developer mode)

3. **"## Writer mode output"**:
   - Writer mode: "Saved. You can see your save history with `/scr:history`."
   - Developer mode: Show full git commit output with hash

4. **"## Edge cases"**:
   - No changes to save: "Nothing new to save. Your last save was [timestamp]."
   - Not in a Scriven project: "No manuscript found. Start with `/scr:new-work`."

**Create `commands/scr/compare.md`** (per D-07, AUTO-08):

Frontmatter:
- `description: Compare your current work with a previous save. Shows changes in plain prose, not code diff.`
- `argument-hint: "[save number or 'last']"`

Body sections:

1. **"# Compare"** -- "You are showing the writer how their prose has changed. Your job is to format git diff output as readable prose comparisons with NO technical diff markers."

2. **"## What to do"**:
   - Check for `.git/` and at least 2 saves. If not: "Need at least two saves to compare."
   - Default: compare current state with last save (HEAD~1)
   - If argument provided: compare with that specific save
   - Run: `git diff HEAD~1 .manuscript/` (or specified ref)
   - Parse diff hunks into changed prose pairs
   - Format as Before/After blocks (per D-07):
     ```
     ## {Unit Name} -- Changes

     **Before:**
     > Marcus walked through the empty corridor, his footsteps echoing.

     **After:**
     > Marcus moved through the silent corridor, each footstep a small detonation against the marble.
     ```

3. **"## Formatting rules"** (per D-07):
   - NEVER show: `+`, `-`, `@@`, line numbers, file paths, commit hashes
   - Use blockquotes (`>`) for prose passages
   - Group changes by unit (chapter/scene) with headings
   - Show unchanged context around changes for readability (1 sentence before/after)
   - If a section was added (not changed): "**New:**" instead of Before/After
   - If a section was removed: "**Removed:**" with the deleted text

4. **"## Developer mode"**: Show standard unified diff with file paths and line numbers

- `description: List your draft versions with readable labels.`
- `argument-hint: "[--all]"`

Body sections:

1. **"# Versions"** -- "You are listing the writer's draft versions. Your job is to show saves as a numbered list with human-readable descriptions."

2. **"## What to do"**:
   - Run: `git log --oneline .manuscript/`
   - Parse commit messages into version list:

     ```
     3. Apr 4 -- Drafted chapter 3 (1,247 words)

     ```
   - Group by date for readability
   - Show word count where available from commit messages
   - Default: last 10 versions. `--all` for complete list.

3. **"## Writer mode output"**: No git hashes, no branch names, numbered list with dates and descriptions.
4. **"## Developer mode"**: Include short hashes, branch info.

**Create `commands/scr/undo.md`** (per D-08, AUTO-10):

Frontmatter:
- `description: Undo your last save and go back to the previous version.`
- `argument-hint: "[--force]"`

Body sections:

1. **"# Undo"** -- "You are reverting the writer's work to the last save point. Your job is to do this safely with explicit confirmation."

2. **"## What to do"**:
   - FIRST: Check for uncommitted/unsaved changes. If found: "You have unsaved changes since your last save. If you undo now, you will lose those changes too. Save first with `/scr:save`, or use `--force` to proceed."
   - Read the last commit message to describe what will be undone
   - Show confirmation prompt (per D-08):
     ```
     This will revert to your last save. You'll lose:
     - Changes from "Saved after drafting chapter 3" (1,247 words)

     Proceed? (yes/no)
     ```
   - On "yes": Run `git reset --soft HEAD~1` (keeps files staged but undoes commit) then `git checkout -- .manuscript/` (restores files to previous state)
   - On "no": "Okay, nothing was changed."
   - Update STATE.md to reflect the reverted position

3. **"## Safety checks"**:
   - Always check for unsaved changes first
   - Never undo past the initial project creation commit
   - If only one save exists: "Nothing to undo. This is your first save."
   - The `--force` flag skips the unsaved changes warning but still shows the confirmation prompt

4. **"## Writer mode output"**: "Undone. You're back to: {previous save description}."
5. **"## Developer mode"**: Show git reset output, hash info.
