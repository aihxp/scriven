Create three commands following established pattern.

**blurb.md** (PUB-04, per D-09):
- YAML frontmatter: `description: Generate marketing blurb in three variations for back cover and retailer listings.`
- Usage: `/scr:blurb`
- Instruction: Role is "marketing copywriter specializing in book blurbs". Load config.json, CONSTRAINTS.json (hidden from academic/speech_song), WORK.md (genre, themes), STYLE-GUIDE.md (match the book's tone), and complete draft
- Per D-09, generate 3 distinct variations with different marketing STRATEGIES (per research pitfall 5, these must NOT be the same text at different lengths):
  1. **Short/Punchy** (50-75 words): Hook + stakes + tagline. Punchy, urgent. Think back-of-book one-liner expanded. Use active voice, present tense. End with a question or cliffhanger.
  2. **Standard** (150-200 words): Setup + conflict + stakes + tone. The classic back-cover blurb. Introduce the protagonist, their world, what disrupts it, what's at stake. End with the central dramatic question.
  3. **Extended** (250-350 words): Detailed setup + character introduction + conflict + themes + comparable title positioning. For retailer listings (Amazon, B&N) where longer descriptions perform better. Include genre positioning and comp titles.
- Present all three to the writer for selection
- Save all three to `.manuscript/marketing/BLURB.md` with clear section headers

**synopsis.md** (PUB-05, per D-10):
- YAML frontmatter: `description: Generate plot synopsis at specified length for query and submission packages.`
- Usage: `/scr:synopsis [--length <1p|2p|5p>]`
- Per D-10, support three lengths:
  - `1p` (1 page, ~500 words): For agent queries. Hit the main plot beats only: protagonist, inciting incident, rising action, climax, resolution. Include the ENDING (synopses always reveal the ending).
  - `2p` (2 pages, ~1000 words): For editor review. All main plot beats plus key subplots and character arcs. Include thematic arc.
  - `5p` (5 pages, ~2500 words): Detailed outline. Chapter-by-chapter or act-by-act summary with character development, subplot tracking, and thematic threads.
- Default to 2p if --length not specified
- Instruction: Role is "publishing synopsis specialist". Load config.json, CONSTRAINTS.json (hidden from academic/poetry/interactive/speech_song/sacred), WORK.md, OUTLINE.md, complete draft
- Critical: "A synopsis ALWAYS reveals the ending. This is not a blurb. Agents and editors need to know how the story resolves."
- Save to `.manuscript/marketing/SYNOPSIS-{length}.md`

**discussion-questions.md** (PUB-08):
- YAML frontmatter: `description: Generate reading group discussion questions exploring themes, characters, and craft.`
- Usage: `/scr:discussion-questions [--count <N>]`
- Default 10-15 questions if --count not specified
- Instruction: Role is "book club facilitator". Load config.json, CONSTRAINTS.json (adapted: study-questions for sacred, hidden from script/academic/visual/poetry/interactive/speech_song), WORK.md, complete draft, STYLE-GUIDE.md
- Question categories: thematic (what does the book say about X?), character (motivations, growth, moral choices), craft (narrative structure, POV choices, symbolism), personal connection (how does this relate to the reader's experience?), debate (questions with no single right answer)
- For sacred work types (adapted as study-questions): Focus on theological themes, scriptural interpretation, practical application, historical context, and interfaith dialogue
- Save to `.manuscript/back-matter/discussion-questions.md`

Create two submission preparation commands.

**query-letter.md** (PUB-06, per D-11):
- YAML frontmatter: `description: Generate agent query letter adapted to genre conventions.`
- Usage: `/scr:query-letter`
- Instruction: Role is "literary agent query specialist". Load config.json, CONSTRAINTS.json (available prose/script/sacred, hidden from academic/visual/poetry/interactive/speech_song), WORK.md (genre, word count, comp titles), complete draft
- Prerequisite check: Verify blurb and synopsis exist in .manuscript/marketing/. If missing, tell writer to run `/scr:blurb` and `/scr:synopsis` first.
- Per D-11, adapt to genre conventions:
  - **Literary fiction**: Lead with voice/prose quality, emphasize themes, use literary comp titles, position within literary tradition
  - **Romance**: Lead with the central relationship, include heat level hint, use romance-specific comp titles, mention tropes
  - **Thriller/Mystery**: Lead with the hook/stakes, emphasize pacing and twists, use genre comp titles, hint at the ticking clock
  - **Fantasy/Sci-Fi**: Lead with the world/concept, mention worldbuilding scope, use genre comp titles, position within subgenre
  - **General fiction**: Balanced approach, lead with character and conflict
  - For other genres: Analyze WORK.md genre field and adapt accordingly
- Standard query structure: Hook (1 sentence), Book paragraph (from blurb, adapted), Bio paragraph (from writer profile), Housekeeping (title, genre, word count, comp titles)
- Save to `.manuscript/marketing/QUERY-LETTER.md`

**book-proposal.md** (PUB-07):
- YAML frontmatter: `description: Generate nonfiction book proposal for agent or publisher submission.`
- Usage: `/scr:book-proposal`
- Instruction: Role is "nonfiction publishing specialist". Load config.json, CONSTRAINTS.json (available prose/sacred, nonfiction_only constraint, hidden from script/academic/visual/poetry/interactive/speech_song), WORK.md, OUTLINE.md, complete draft
- Check nonfiction_only constraint: If work type is fiction (novel, short_story, etc.), tell writer "Book proposals are for nonfiction works. Fiction is queried with a completed manuscript and query letter (/scr:query-letter)."
- Prerequisite: Synopsis must exist. If missing, tell writer to run `/scr:synopsis` first.
- Proposal sections:
  1. Overview -- What the book is about and why it matters now (market timing)
  2. Target audience -- Specific reader profile with market data
  3. Competitive analysis -- How this book differs from existing titles
  4. Author platform -- From writer profile, relevant expertise, reach
  5. Marketing plan -- Author's promotional capabilities and ideas
  6. Chapter outline -- From OUTLINE.md with expanded descriptions
  7. Sample chapters -- Reference to existing drafted chapters
  8. Timeline -- Estimated completion/delivery timeline
- Save to `.manuscript/marketing/BOOK-PROPOSAL.md`
