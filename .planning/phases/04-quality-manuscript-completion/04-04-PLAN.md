**front-matter.md**:
- YAML frontmatter: `description: Generate publication-ready front matter elements in Chicago Manual of Style order.`
- Usage section:
  ```
  /scr:front-matter [--element <name>]
  ```
  Elements: half-title, series-title, title, copyright, dedication, epigraph, toc, illustrations-list, tables-list, abbreviations, foreword, preface, acknowledgments, introduction, prologue, note-to-reader, maps, cast, timeline
  Per D-07: If --element is provided, generate only that element. Otherwise generate all applicable elements.

- Instruction: Role is "publishing specialist". Load config.json, CONSTRAINTS.json, WORK.md (title, author, ISBN, publisher info), OUTLINE.md (for TOC generation), CHARACTERS.md (for cast of characters)

- Per D-06, separate Voice DNA handling:
  - Load STYLE-GUIDE.md for narrative elements: dedication, epigraph (suggestions), preface, acknowledgments, introduction, prologue, note-to-reader
  - Do NOT load STYLE-GUIDE.md for mechanical elements: half-title, series-title, title page, copyright, toc, illustrations-list, tables-list, abbreviations, foreword scaffold, maps, cast, timeline

- All 19 elements in Chicago Manual of Style order with generation type:
  1. Half-title (Recto) -- GENERATE: title only, no subtitle or author
  2. Series title / Also by (Verso) -- GENERATE: from series info in WORK.md
  3. Title page (Recto) -- GENERATE: title, subtitle, author, publisher
  4. Copyright page (Verso) -- GENERATE: template with ISBN, copyright year, publisher, rights statement, edition info. Include standard legal boilerplate
  5. Dedication (Recto) -- SCAFFOLD: provide template with guidance, writer fills in. Load STYLE-GUIDE.md for tone
  6. Epigraph (Recto/Verso) -- SUGGEST: offer 3 thematically appropriate options from public domain, writer picks or supplies own. Load STYLE-GUIDE.md for tone matching
  7. Table of contents (Recto) -- GENERATE: from OUTLINE.md structure
  8. List of illustrations (Recto) -- GENERATE: from manuscript content if illustrations exist
  9. List of tables (Recto) -- GENERATE: from manuscript content if tables exist
  10. List of abbreviations (Recto) -- GENERATE: extract abbreviations used in manuscript
  11. Foreword (Recto) -- SCAFFOLD: template with instructions for the guest writer. Note: "A foreword is written by someone other than the author. This scaffold provides structure and guidance for your foreword writer." Do NOT generate AI content pretending to be from another person (per research pitfall 3)
  12. Preface (Recto) -- SCAFFOLD: template with prompts for the author. Load STYLE-GUIDE.md
  13. Acknowledgments (Recto) -- GENERATE DRAFT: from project context (collaborators, influences). Load STYLE-GUIDE.md. Note: "This is a draft — personalize with your own thanks"
  14. Introduction (Recto) -- SCAFFOLD: template with section prompts. Load STYLE-GUIDE.md
  15. Prologue (Recto) -- SCAFFOLD: narrative opening template. Load STYLE-GUIDE.md
  16. Note to the reader (Variable) -- GENERATE DRAFT: context-appropriate note. Load STYLE-GUIDE.md
  17. Maps / family trees (Variable) -- SCAFFOLD: prompt for map creation from WORLD.md if it exists
  18. Cast of characters (Variable) -- GENERATE: from CHARACTERS.md
  19. Timeline (Variable) -- GENERATE: from timeline data if available

- Per D-08 academic adaptation (behavior: academic_front_matter): Replace dedication/epigraph with abstract. Add list of figures. Emphasis on methodology introduction.
- Per D-08 sacred adaptation (behavior: sacred_front_matter): Add imprimatur/nihil obstat scaffold. Add tradition-specific greeting. Emphasis on textual provenance.

- When running without --element, only generate elements applicable to the work type. List skipped elements at the end with brief explanation of why they were skipped.

- Save each element to `.manuscript/front-matter/{NN}-{element-name}.md` using the Chicago Manual order number as prefix for correct sequencing

- YAML frontmatter: `description: Generate publication-ready back matter elements.`
- Usage section:
  ```

  ```
  Elements: epilogue, afterword, appendix, glossary, endnotes, bibliography, suggested-reading, index, about-author, colophon, discussion-questions, permissions

- Instruction: Role is "publishing specialist". Load config.json, CONSTRAINTS.json, WORK.md, CHARACTERS.md (for glossary extraction), and drafted prose

- Per D-06 Voice DNA handling:
  - Load STYLE-GUIDE.md for: epilogue scaffold, afterword scaffold, suggested-reading, about-author, discussion-questions
  - Do NOT load for: appendix, glossary, endnotes, bibliography, index, colophon, permissions

- All 12+ elements with generation type:
  1. Epilogue -- SCAFFOLD: narrative closing template. Load STYLE-GUIDE.md
  2. Afterword -- SCAFFOLD: author reflection template. Load STYLE-GUIDE.md
  3. Appendix(es) -- TEMPLATE: structured appendix template for supplementary material
  4. Glossary -- GENERATE: extract terms, names, and concepts from manuscript with definitions
  5. Endnotes -- TEMPLATE: numbered endnote structure from any footnotes in manuscript
  6. Bibliography / References -- TEMPLATE: bibliographic entry format per work type
  7. Suggested reading -- GENERATE: curated reading list from themes and influences. Load STYLE-GUIDE.md for tone
  8. Index -- TEMPLATE: provide indexing instructions and key term extraction. Note: "Professional indexing is recommended for nonfiction. This provides a starting framework."
  9. About the author -- GENERATE: from writer profile if available. Load STYLE-GUIDE.md for voice. Draft in third person unless writer profile specifies otherwise
  10. Colophon -- TEMPLATE: production details (fonts, tools, printing info)
  11. Discussion questions -- GENERATE: reading group questions from themes and plot. Load STYLE-GUIDE.md. (Also available as standalone /scr:discussion-questions)
  12. Permissions / Credits -- TEMPLATE: permissions acknowledgment structure for quoted material, images, etc.

- Per D-08 academic adaptation (behavior: academic_back_matter): Bibliography becomes primary element (APA/MLA/Chicago format based on config). Add methodology appendix template. Index gets enhanced term extraction.
- Per D-08 sacred adaptation (behavior: sacred_back_matter): Add concordance scaffold. Add maps of biblical/historical lands. Add chronological tables. Glossary emphasizes theological/tradition terms.

- When running without --element, only generate applicable elements. List skipped elements at end.
- Save each element to `.manuscript/back-matter/{element-name}.md`
