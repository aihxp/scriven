  <action>
Create two quality review command files following the established command pattern (YAML frontmatter with description, Usage section, Instruction section, structured output).

**line-edit.md** (QUAL-01, per D-01):
- YAML frontmatter: `description: Perform a line-level prose quality pass with inline annotations.`
- Usage: `/scr:line-edit [N]` -- N scopes to specific unit, omit for full manuscript
- Instruction: Role is "line editor". Load config.json, CONSTRAINTS.json, STYLE-GUIDE.md, and drafted prose from .manuscript/drafts/
- Analysis must be grouped by four categories per D-01: rhythm (sentence length variation, cadence, paragraph flow), word_choice (weak verbs, imprecise nouns, register mismatches), redundancy (repeated information, unnecessary modifiers, filler phrases), cliches (dead metaphors, overused phrases, genre stock phrases)
- Output format per D-01: For each issue show Original -> Suggested replacement, the Type (rhythm|word_choice|redundancy|cliche), and brief Rationale
- Explicitly instruct: "Preserve the writer's voice -- suggest improvements that sound like a better version of THIS writer, not generic polished prose. Load STYLE-GUIDE.md to understand the target voice."
- Save to `.manuscript/{scope}-LINE-EDIT-REPORT.md`

**copy-edit.md** (QUAL-02):
- YAML frontmatter: `description: Perform a correctness pass for grammar, spelling, punctuation, and consistency.`
- Usage: `/scr:copy-edit [N]`
- Instruction: Role is "copy editor". Load config.json, CONSTRAINTS.json, and drafted prose
- Check categories: grammar (subject-verb agreement, tense consistency, dangling modifiers), spelling (typos, homophones, proper nouns), punctuation (comma usage, dialogue punctuation, em-dashes vs hyphens), consistency (character name spelling, place name spelling, terminology usage, hyphenation choices, number style)
- Do NOT load STYLE-GUIDE.md for voice -- copy editing is mechanical, not stylistic
- Output: For each issue show the Original text, the Correction, the Category, and Location reference
- Save to `.manuscript/{scope}-COPY-EDIT-REPORT.md`
