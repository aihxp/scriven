<objective>
Fix REWRITE-01 and REWRITE-02: installed Codex command files must use `$scr-*` invocation syntax (not raw `/scr:*`), and the shared rewriter must leave references inside fenced code blocks unchanged so documentation examples remain portable across runtimes.

Purpose: Today `installCodexRuntime` `copyDir`s `commands/scr/**` verbatim into `.codex/commands/scr/*.md`, so Codex users see `/scr:help` which does not invoke anything in their host. Simultaneously, `rewriteInstalledCommandRefs` uses a global regex that mangles documentation examples of `/scr:*` that live inside code fences. Both are narrow correctness bugs in the installer.

Output: Updated `bin/install.js` with a code-block-aware rewriter, a new exported `generateCodexCommandContent`, and a refactored Codex install path; plus regression tests in `test/install.test.js`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| package source → user filesystem | Installer reads `PKG_ROOT/commands/scr/**` (trusted, shipped with npm package) and writes to `~/.codex/commands/scr/**` or `.codex/commands/scr/**` in the user's project. No untrusted input crosses this boundary — all command markdown is authored in-repo. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-27-01 | Tampering | `.codex/commands/scr/*.md` mid-install | mitigate | Use `atomicWriteFileSync` (temp-then-rename) for every rewritten file — same pattern already used for SAFE-01. No partial writes visible to Codex. |
| T-27-02 | Denial of Service | Unterminated fence in a malformed command file could cause infinite loop or OOM in the segmenter | mitigate | Segmenter is a single linear pass over `content.split(/\r?\n/)` with O(n) time and no backtracking; unterminated fence is explicitly handled as "rest is code" fail-safe. |
| T-27-03 | Information Disclosure | Rewriter accidentally leaks installer internals into output | accept | Transform functions are pure string replacements over `/scr:[a-z0-9:-]+`; no env vars, paths, or secrets are consulted. Low risk, no PII. |
| T-27-04 | Tampering | `.tmp.*` files orphaned on crash in `.codex/commands/scr/` | mitigate | `cleanOrphanedTempFiles` (SAFE-02) already runs at installer startup and will sweep this directory tree on next run. No new mitigation required beyond ensuring writes go through `atomicWriteFileSync`. |
</threat_model>
