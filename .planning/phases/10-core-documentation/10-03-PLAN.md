  <action>
Create `docs/command-reference.md`. Per D-03: one entry per command with description, usage, flags, example, prerequisites, work-type adaptations.

**Process:**

1. Read `data/CONSTRAINTS.json` completely. Extract from the `commands` dict:
   - Command name
   - Description
   - Category
   - Prerequisites (what must exist before this command works)
   - Work-type availability/restrictions
   - Any flags or arguments

2. Read the `command_adaptations` section of CONSTRAINTS.json to understand how command names change per work type (e.g., draft-chapter becomes draft-surah for Quran).

3. For each of the 14 categories in CONSTRAINTS.json, create a section. Use this order (logical flow for a writer):
   - Core (new-work, discuss, plan, draft, etc.)
   - Navigation (next, do, help, etc.)
   - Session (save, history, compare, versions, etc.)
   - Structure (plot-graph, timeline, theme-tracker, etc.)
   - Structure Management (add-unit, insert-unit, remove-unit, etc.)
   - Character & World (new-character, character-sheet, build-world, etc.)
   - Quality (voice-test, line-edit, copy-edit, polish)
   - Review (continuity-check, voice-check, sensitivity-review, etc.)
   - Publishing (front-matter, back-matter, blurb, synopsis, etc.)
   - Illustration (cover-art, illustrate-scene, character-ref, etc.)
   - Translation (translate, translation-glossary, cultural-adaptation, etc.)
   - Collaboration (track)
   - Utility (manager, health, fast, add-note, etc.)
   - Sacred Exclusive (concordance, cross-reference, genealogy, etc.)

4. For each command entry, format as:

```
### `/scr:command-name`

**Description:** [from CONSTRAINTS.json]

**Usage:** `/scr:command-name [arguments]`

**Prerequisites:** [what must exist — e.g., "OUTLINE.md must exist", or "None"]

**Work-type adaptations:** [if the command name changes — e.g., "becomes `/scr:draft-surah` for Quran commentary"]

**Example:**
[A brief, practical example of using the command]
```

5. For the top of the document, include:
   - Title: "Command Reference"
   - Brief intro: "Scriven has 101 commands organized into 14 categories..."
   - Table of contents linking to each category section
   - Note about work-type adaptations: commands auto-rename based on your work type

6. For EACH command, also read its corresponding command file (`commands/scr/{name}.md`) to extract the `description` and `argument-hint` from frontmatter. For sacred-exclusive commands, read from `commands/scr/sacred/{name}.md`.

IMPORTANT: Do NOT invent commands or flags. Every entry must come from CONSTRAINTS.json or the command files. If a command file doesn't have flag details, document what's available without fabricating.

Per D-01: tone is friendly and practical. Each example should be something a writer would actually do, not abstract.
