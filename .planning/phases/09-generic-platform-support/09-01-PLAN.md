<objective>
Refactor the installer to support two installation strategies: command-directory (existing) and skill-file (new). Add a `type` field to every RUNTIMES entry, create the SKILL.md generation function, refactor Manus to use the skill-file path, and add a "Generic (SKILL.md)" fallback option.

Purpose: Enable Scriven installation on platforms that lack `.claude/commands/`-style directories by providing a consolidated SKILL.md manifest that any AI agent can discover.

Output: Updated `bin/install.js` with type-based routing, SKILL.md generation, and Generic option.
</objective>

  <action>
Modify bin/install.js with the following changes:

1. **Add `type` field to every RUNTIMES entry** (per D-01, PLAT-01):
   - All existing 8 command-directory runtimes (claude-code, cursor, gemini-cli, codex, opencode, copilot, windsurf, antigravity) get `type: 'commands'`
   - Manus gets `type: 'skills'` (per D-02, PLAT-03)
   - For skill-file runtimes, replace the four `commands_dir_*` / `agents_dir_*` properties with:
     - `skills_dir_global`: base directory for skills (e.g., `path.join(os.homedir(), '.manus', 'skills', 'scriven')`)
     - `skills_dir_project`: project-relative equivalent (e.g., `.manus/skills/scriven`)
   - Keep `detect()` on Manus as-is: checks `~/.manus/` or `Manus.app`

2. **Add a 'generic' runtime entry** as the LAST entry in RUNTIMES (per D-03, PLAT-04):
   ```javascript
   'generic': {
     label: 'Generic (SKILL.md)',
     type: 'skills',
     skills_dir_global: path.join(os.homedir(), '.scriven', 'skills'),
     skills_dir_project: '.scriven/skills',
     detect: () => false, // never auto-detected; user selects manually
   }
   ```

3. **Create `generateSkillManifest(constraintsPath)` function** (per D-04, PLAT-05):
   - Reads CONSTRAINTS.json from the given path
   - Iterates `constraints.commands` to build a markdown manifest
   - Returns a string with this format:
     ```
     # Scriven — AI Creative Writing Skills

     Version: {version from package.json}

     Scriven is a spec-driven creative writing, publishing, and translation pipeline.

     ## Available Commands

     | Command | Category | Description |
     |---------|----------|-------------|
     | `/scr:new-work` | core | Initialize a new creative work |
     | `/scr:draft` | core | Draft the planned unit |
     ...

     ## Usage

     Each command above has a detailed instruction file in the `commands/scr/` subdirectory.
     To use a command, read the corresponding `.md` file and follow its instructions.

     ## Quick Start

     1. Run `/scr:help` to see commands grouped by stage
     2. Run `/scr:new-work` to start a new project
     3. Run `/scr:demo` to explore a sample project
     ```
   - Sort commands by category, then alphabetically within category
   - Include sacred subcommands (from `commands/scr/sacred/`) as `/scr:sacred:{name}`

4. **Branch the install flow in `main()` based on `runtime.type`**:
   - If `runtime.type === 'commands'` (or type is undefined for backward compat): run existing install logic unchanged
   - If `runtime.type === 'skills'`:
     - Determine `skillsDir` from `isGlobal ? runtime.skills_dir_global : path.resolve(runtime.skills_dir_project)`
     - Determine `dataDir` as before (unchanged)
     - Call `generateSkillManifest(path.join(PKG_ROOT, 'data', 'CONSTRAINTS.json'))` and write result to `path.join(skillsDir, 'SKILL.md')`
     - Copy `commands/scr/` to `path.join(skillsDir, 'commands', 'scr')` (per D-05 — individual command files alongside manifest)
     - Copy `agents/` to `path.join(skillsDir, 'agents')`
     - Copy templates and data to `dataDir` as before
     - Write `settings.json` as before
     - Print success messages adapted for skill-file path

5. **Export `generateSkillManifest`** alongside `copyDir` and `RUNTIMES` in `module.exports`.
