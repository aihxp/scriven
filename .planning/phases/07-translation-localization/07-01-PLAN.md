Create `agents/translator.md` mirroring the drafter agent pattern (per D-01):

**Translator agent receives per unit:**
1. **Source text** — the original atomic unit being translated (one scene/subsection/passage/stanza)
2. **STYLE-GUIDE.md** — voice DNA of the source to understand authorial intent and register
3. **GLOSSARY-{lang}.md** — approved translations for character names, place names, invented terms, idioms, recurring phrases
4. **Translation memory excerpt** — relevant segments from translation-memory.json (TM matches for this unit's content)
5. **CHARACTERS.md excerpt** — relevant characters for dialogue voice preservation in translation
6. **Previous translated unit tail** — last 200 words of the previously translated unit for flow continuity in the target language
7. **Target language + config** — language code, name_handling setting (keep_original/transliterate/localize), measurement_system setting

**Translator agent behavior:**
- Step 1: Load and read all provided files. Understand STYLE-GUIDE.md deeply — authorial voice must be preserved in translation
- Step 2: Orient — identify register (formal, colloquial, literary), dialogue voices, emotional tone
- Step 3: Consult glossary — every character name, place name, and invented term MUST use the approved translation from GLOSSARY-{lang}.md. If a term appears that has no glossary entry, translate it and flag it as "NEW TERM — add to glossary"
- Step 4: Consult translation memory — reuse approved segments from TM where they match
- Step 5: Translate — preserve voice, tone, register. Dialogue must reflect each character's speech patterns in the target language. Adapt idioms rather than translate literally (flag adaptations)
- Step 6: Self-check — are all glossary terms used correctly? Is the register consistent? Does each character still sound distinct? Any untranslatable concepts that need footnotes?
- Step 7: Write to `.manuscript/translation/{lang}/drafts/{unit}-DRAFT.md`

**What translator must never do:**
- Translate word-for-word at the expense of natural target language flow
- Change the meaning, tone, or emotional arc of the original
- Use a glossary term inconsistently
- Translate in a voice that doesn't match the source STYLE-GUIDE.md intent

Create `commands/scr/translate.md` with:

**Frontmatter:**
```yaml
description: Translate manuscript to target language with glossary and translation memory support.
argument-hint: "<language> [--all] [--from <unit>] [--languages] [--add-language <lang>]"
```

**Command behavior:**
1. Read `.manuscript/config.json` for source_language, target_languages, name_handling, measurement_system
2. If `--languages` flag: list configured target_languages and exit
3. If `--add-language <lang>` flag: add language to config.json target_languages array and exit
4. Validate target language is in target_languages list (if not, suggest --add-language)
5. Check prerequisite: complete-draft exists (all units in OUTLINE.md have drafts)
6. Read `.manuscript/OUTLINE.md` for unit list and order
7. If `--from <unit>` flag: start from specified unit (for resuming)
8. If `--all` flag: translate all configured target_languages sequentially
9. For each unit in OUTLINE.md:
   a. Load source text from `.manuscript/drafts/{unit}-DRAFT.md`
   b. Load GLOSSARY-{lang}.md if it exists (recommend creating one first if missing)
   c. Load relevant translation memory segments from translation-memory.json
   d. Invoke translator agent with fresh context (source text + glossary + TM + STYLE-GUIDE.md + CHARACTERS.md excerpt)
   e. Agent writes translated unit to `.manuscript/translation/{lang}/drafts/{unit}-DRAFT.md`
   f. After each unit: update progress, report word count, flag any new terms not in glossary
10. After all units: show summary (units translated, word counts source vs target, new glossary terms flagged, TM segments reused)
11. Create `.manuscript/translation/{lang}/` directory structure: drafts/, front-matter/, back-matter/, metadata/

Create `commands/scr/translation-glossary.md` (per D-02):

**Frontmatter:**
```yaml
description: Create and manage bilingual term glossary for consistent translation.
argument-hint: "<language> [--add <term> --translation <value>] [--import] [--review] [--category <cat>]"
```

**Command behavior:**
1. Read `.manuscript/config.json` for target_languages
2. If no language specified, prompt for one from target_languages
3. **Create mode** (no GLOSSARY-{lang}.md exists): Scan manuscript for translatable terms:
   - Character names from CHARACTERS.md (or FIGURES.md for sacred works)
   - Place names from WORLD.md (if exists)
   - Invented terms, recurring phrases, titles/honorifics
   - For each term, offer 3 columns in markdown table: Source Term | Translation | Category | Notes
   - Categories: character_name, place_name, invented_term, title_honorific, recurring_phrase, cultural_reference, brand_name
   - For name_handling=keep_original: default character names to original with transliteration in Notes
   - For name_handling=transliterate: provide transliteration
   - For name_handling=localize: provide localized equivalent
   - Write to `.manuscript/translation/GLOSSARY-{lang}.md` as markdown table (per D-02: human-readable, version-controlled)
4. **Add mode** (`--add <term> --translation <value>`): Append entry to existing glossary table
5. **Import mode** (`--import`): Bulk import from a list of terms provided by the writer
6. **Review mode** (`--review`): Show all glossary entries, highlight any terms found in manuscript but missing from glossary
7. Glossary format (markdown table):
```markdown
# Translation Glossary: {Language}

| Source Term | Translation | Category | Notes |
|-------------|-------------|----------|-------|
| Marcus | マルクス | character_name | Transliteration |
```

Create `commands/scr/translation-memory.md`:

**Frontmatter:**
```yaml
description: Build and manage translation memory from completed translations.
argument-hint: "<language> [--build] [--stats] [--export] [--clear]"
```

**Command behavior:**
1. Read `.manuscript/config.json` for target_languages
2. **Build mode** (`--build` or default when TM doesn't exist):
   - Scan `.manuscript/translation/{lang}/drafts/` for completed translation drafts
   - Align source segments (sentences) with translated segments using paragraph/sentence boundaries
   - Store as JSON array of segment pairs in `.manuscript/translation/translation-memory.json`:
     ```json
     {
       "source_language": "en",
       "entries": {
         "fr": [
           {"source": "...", "target": "...", "unit": "01-01", "confidence": 1.0}
         ]
       }
     }
     ```
   - Deduplicate exact matches, keep highest confidence
3. **Stats mode** (`--stats`): Show TM statistics — total segments, per-language breakdown, coverage percentage
4. **Export mode** (`--export`): Export TM as TMX (Translation Memory eXchange) format for external tools
5. **Clear mode** (`--clear`): Clear TM for a specific language (with confirmation)
6. TM is referenced by translate command: when translating a new unit, relevant TM segments are loaded into translator agent context
