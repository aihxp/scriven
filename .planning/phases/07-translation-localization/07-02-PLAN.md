Create `commands/scr/cultural-adaptation.md`:

**Frontmatter:**
```yaml
description: Flag idioms, humor, customs, and cultural references that need localization for target language.
argument-hint: "<language> [--unit <unit>] [--severity <level>] [--report]"
```

**Command behavior:**
1. Read `.manuscript/config.json` for source_language and target_languages
2. Validate language argument is in target_languages
3. Check prerequisite: translation exists for this language (`.manuscript/translation/{lang}/drafts/` has files)
4. Scan each translated unit (or specific `--unit`) for cultural adaptation needs
5. Flag categories with severity levels (high, medium, low):

| Category | Severity | Examples |
|----------|----------|----------|
| idiom | high | "break a leg", "raining cats and dogs" — meaning lost in literal translation |
| humor | high | Culturally-specific jokes, wordplay, puns |
| custom | medium | Holidays, social norms, greeting conventions |
| measurement | medium | Miles/km, Fahrenheit/Celsius, cups/ml |
| currency | medium | Dollar amounts, pricing conventions |
| food | low | Cultural food references, meal customs |
| name_order | medium | Family name first/last conventions |
| politeness | high | T-V distinction (tu/vous, du/Sie), honorific levels (Japanese keigo) |
| punctuation | low | Quotation mark style, spacing conventions |

6. For each flag, provide:
   - Source text (original)
   - Current translation
   - Issue description
   - Suggested adaptation (or "needs manual review")
   - Severity level
7. If `--report` flag: generate a standalone adaptation report at `.manuscript/translation/{lang}/CULTURAL-ADAPTATION-REPORT.md`
8. If `--severity <level>` flag: filter to show only flags at or above specified severity

Create `commands/scr/back-translate.md` (per D-03):

**Frontmatter:**
```yaml
description: Translate the translation back to source language and show side-by-side comparison with drift annotations.
argument-hint: "<language> [--unit <unit>] [--report]"
```

**Command behavior:**
1. Read `.manuscript/config.json` for source_language
2. Validate language argument and check translation exists
3. For each translated unit (or specific `--unit`):
   a. Load original source text from `.manuscript/drafts/{unit}-DRAFT.md`
   b. Load translated text from `.manuscript/translation/{lang}/drafts/{unit}-DRAFT.md`
   c. Translate the translation back to source language (AI agent does this in-context, no external API needed)
   d. Show three-column side-by-side comparison per D-03:
      ```
      | Original | Translation | Back-Translation |
      |----------|-------------|------------------|
      ```
   e. Add drift annotations highlighting meaning changes:
      - [DRIFT: meaning shift] — back-translation differs semantically from original
      - [DRIFT: tone shift] — emotional register changed
      - [DRIFT: omission] — content present in original missing in back-translation
      - [DRIFT: addition] — content in back-translation not in original
      - [OK] — meaning preserved
4. Summary statistics: total segments, OK count, drift count by type
5. If `--report` flag: write report to `.manuscript/translation/{lang}/BACK-TRANSLATION-REPORT.md`
  </action>

  </acceptance_criteria>

Create `commands/scr/multi-publish.md`:

**Frontmatter:**
```yaml
description: Export translated editions in all target formats with localized front/back matter and language-specific formatting.
argument-hint: "[--languages <lang1,lang2,...>] [--format <format>] [--all-languages] [--all-formats]"
```

**Command behavior:**
1. Read `.manuscript/config.json` for target_languages, source_language, work_type
2. Check prerequisites: translations exist for requested languages, export pipeline functional
3. If no flags: prompt for languages and format selection
4. If `--all-languages`: export all languages in target_languages
5. If `--all-formats`: export all available formats for the work type
6. For each language:
   a. **Localize front matter:** Generate translated front matter elements:
      - Title page (translated title, subtitle, author name in target script if applicable)
      - Copyright page (translated copyright notice, translation credit, ISBN if provided)
      - Dedication, epigraph (translate or preserve in source per writer preference)
      - Table of contents (translated chapter/section titles)
   b. **Localize back matter:** Generate translated back matter:
      - About the author (translated bio)
      - Also by (with local marketplace ISBNs if configured)
      - Acknowledgments (translated)
      - Translator acknowledgment (auto-generated credit line)
   c. **Language-specific formatting:**
      - Quotation marks: "" for English, "" for French, "" for German, "" for Japanese, etc.
      - Punctuation spacing: French thin space before ;:!?, no extra space for English
      - Text direction: ltr or rtl based on language (handled by Plan 03 in detail)
      - Number formatting: decimal comma vs period per locale
   d. **Compile:** Assemble front matter + translated body + back matter into single document
   e. **Export:** Chain to export command with language-specific metadata:
      - Output to `.manuscript/output/translations/{lang}/`
      - Filename includes language code: `manuscript-{lang}.{ext}`
      - Pass language-specific Pandoc variables (lang, dir, quotes)
   f. **Package:** If format is kdp-package or ingram-package, create language-specific package
7. Show summary: languages exported, formats per language, output paths, total file count
