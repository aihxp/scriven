Create `commands/scr/autopilot-translate.md` (per D-05):

**Frontmatter:**
```yaml
description: Run multi-language translation pipeline unattended with glossary, translation, adaptation, and publishing.
argument-hint: "[--languages <lang1,lang2,...>] [--all-languages] [--skip-publish] [--skip-adaptation] [--resume]"
```

**Command behavior:**
1. Read `.manuscript/config.json` for target_languages, source_language
2. Determine languages to process: `--languages` flag, `--all-languages`, or prompt
3. Read `.manuscript/OUTLINE.md` for unit count
4. Read `.manuscript/STATE.md` for resume position (if `--resume`)

**Pipeline per language (per D-05: parallel agents per language):**

For each target language (conceptually parallel -- each language is independent):

a. **Phase 1: Glossary** — If GLOSSARY-{lang}.md doesn't exist, run translation-glossary to create it. If it exists, skip.
b. **Phase 2: Translate** — Run translate command for all units. Track progress per unit.
c. **Phase 3: Translation Memory** — Run translation-memory --build to capture segments from completed translation.
d. **Phase 4: Cultural Adaptation** (unless `--skip-adaptation`) — Run cultural-adaptation to flag issues. Log flagged items count.
e. **Phase 5: Back-Translate** — Run back-translate for quality verification. Log drift count.
f. **Phase 6: Multi-Publish** (unless `--skip-publish`) — Run multi-publish for this language in all available formats.

**Progress tracking:**
- Update STATE.md after each phase per language
- Show progress: "[lang] Phase 2/6: Translating unit 5/12..."
- If interrupted, `--resume` picks up at last completed phase/unit

**Completion summary:**
- Per language: units translated, word count, glossary terms, TM segments, cultural flags, drift annotations, formats exported
- Overall: total languages, total words translated, total files exported

**RTL detection (per D-04):**
When processing a language, auto-detect if RTL:
```
RTL_LANGUAGES = ["ar", "he", "ur", "fa", "yi", "ps", "sd"]
CJK_LANGUAGES = ["zh", "ja", "ko"]
```
Pass appropriate flags to export: `--text-direction rtl` for RTL languages.

**Update `data/export-templates/scriven-book.typst`:**

The Typst template already has `text-dir` variable. Enhance it to properly apply RTL:
- Add `#set text(dir: text-dir)` to apply text direction to body
- Add `#set par(leading: ...)` with CJK-appropriate line spacing when text-dir is detected as CJK
- Ensure page numbering and headers flip for RTL (inside margin becomes right margin)
- Add CJK font fallback: when language is zh/ja/ko, add appropriate font family (Noto Sans CJK as fallback)
- Keep all existing template functionality intact

**Update `data/export-templates/scriven-epub.css`:**

Add RTL and CJK CSS rules (the existing CSS uses em/rem per Phase 5 decision):
- Add `[dir="rtl"]` selector rules: `direction: rtl; unicode-bidi: embed;`
- Add CJK `writing-mode` rule for vertical text option: `.vertical-cjk { writing-mode: vertical-rl; }`
- Add CJK line break rules: `word-break: break-all;` for CJK, `line-break: strict;` for Japanese
- Add CJK punctuation: `text-align: justify; text-justify: inter-ideograph;` for CJK
- Add RTL-specific list styling (right-aligned bullets/numbers)
- Keep all existing CSS intact — append new rules at the end in clearly marked sections
