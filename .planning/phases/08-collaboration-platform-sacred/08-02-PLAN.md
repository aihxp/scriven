Update `commands/scr/editor-review.md` to support the full editor-writer collaboration workflow (COLLAB-07). The existing editor-review command handles single-reviewer feedback. Extend it with collaboration-aware modes.

**Add `--proposal <name>` flag** for reviewing a revision proposal:
- Reads `.manuscript/proposals/{slug}-proposal.md` created by `track propose`
- Displays the proposal's changes passage by passage
- For each changed passage, editor can:
  - **Accept** — mark change as approved
  - **Reject** — mark change as rejected with reason
  - **Modify** — accept with editor's modification
  - **Note** — add a comment without accepting/rejecting
- All decisions stored in `.manuscript/proposals/{slug}-decisions.json`:
  ```json
  {
    "proposal": "slug",
    "reviewer": "editor",
    "date": "ISO date",
    "decisions": [
      { "passage": "path/to/file:line", "action": "accept|reject|modify|note", "reason": "...", "modification": "..." }
    ],
    "summary": { "accepted": N, "rejected": N, "modified": N, "noted": N }
  }
  ```
- After all passages reviewed, generate summary: "Review complete: X accepted, Y rejected, Z modified, W notes added."

**Add `--notes` flag** for adding editor notes to the current track:
- Editor selects specific passages (by unit number and approximate location)
- Adds inline notes stored in `.manuscript/editor-notes/{unit}-notes.md`
- Note format: `[EDITOR NOTE @ passage]: comment text`
- Notes are visible to the writer but do not change the manuscript text

**Add `--respond` flag** for writer to review editor decisions:
- Shows all editor decisions from the proposal
- Writer can accept each decision or push back with explanation
- Final decisions stored alongside editor decisions for full accountability trail
- After all decisions processed: apply accepted changes via merge, skip rejected ones

**Decision tracking for accountability:**
- Every accept/reject/modify is timestamped and attributed
- Full decision history preserved in proposals directory
- Neither party's changes are lost -- rejected changes remain in the track for reference

**Update `commands/scr/track.md`** to add co-writing support (COLLAB-08):

Add a section "## Co-Writing Parallel Tracks" after the existing subcommands:

When `track merge` detects that the merging track and canon have BOTH been modified (i.e., parallel work happened), automatically:
1. Run continuity-check on the combined content before finalizing merge
2. Surface any contradictions found:
   - Character state conflicts (character alive in one track, dead in another)
   - Timeline conflicts (events happening simultaneously in different locations with the same character)
   - World-state conflicts (world-building facts that contradict)
3. Present each contradiction with context from both tracks
4. Writer resolves each: pick one, reconcile manually, or flag for later
5. Log all resolutions in `.manuscript/merge-log.json` with both versions preserved

Add `--co-writing` flag to `track create`:
- When creating a track with `--co-writing`, mark it in tracks.json with `"type": "co-writing"` (vs default `"revision"`)
- Co-writing tracks show a warning before merge: "This is a co-writing track. Continuity checking will be thorough -- this may take a moment."
