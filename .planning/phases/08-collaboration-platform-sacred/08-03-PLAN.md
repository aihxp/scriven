**`commands/scr/manager.md` (RUNTIME-03, per D-04):**
```yaml
description: Interactive command center for managing multiple writing projects.
argument-hint: "[--list] [--switch <project>] [--status]"
```
- `--list` (default): Scan current directory and subdirectories for `.manuscript/` folders. For each, read `config.json` to get work type, `WORK.md` for title, `STATE.md` for progress. Display table: Project Name | Work Type | Progress | Last Activity
- `--switch <project>`: Change working directory context to that project
- `--status`: Show detailed status of current project (word count, units drafted/total, last save, current track)
- If no `.manuscript/` found: "No Scriven projects found in this directory. Start one with /scr:new-work"

**`commands/scr/health.md` (RUNTIME-05):**
```yaml
description: Diagnose and repair common project state issues.
argument-hint: "[--repair]"
```
- Without `--repair`: Diagnostic mode. Check for:
  1. Missing required files (WORK.md, OUTLINE.md, STYLE-GUIDE.md, config.json, STATE.md)
  2. config.json schema validity (all required fields present)
  3. STATE.md consistency (does progress match actual drafted files?)
  4. Orphaned drafts (draft files not referenced in OUTLINE.md)
  5. Git state (uncommitted changes, detached HEAD)
  6. CONSTRAINTS.json integrity (all commands referenced exist as files)
- With `--repair`: Fix what can be auto-fixed:
  1. Regenerate missing STATE.md from file system state
  2. Fix config.json missing fields with defaults
  3. Report orphaned drafts for manual review
  4. Suggest git commands for git issues
- Output: health report with green/yellow/red indicators per check

**`commands/scr/fast.md` (RUNTIME-07):**
```yaml
description: Make a quick inline edit without full planning overhead. For small fixes and tweaks.
argument-hint: "<description of edit>"
```
- Takes a natural language description of a small edit (e.g., "change Marcus's eye color to brown in chapter 3", "fix the typo in scene 2 paragraph 4")
- Reads the relevant file, makes the edit directly, saves
- No discuss/plan/draft cycle -- just read, edit, write
- Limits: refuse if the edit would change more than ~500 words or touch more than 2 files. Suggest `/scr:do` instead.
- Auto-saves via `/scr:save` pattern after edit
- Confirm what was changed: "Changed Marcus's eye color from blue to brown in chapter-03/scene-01.md"

**`commands/scr/add-note.md` (RUNTIME-06):**
```yaml
description: Add a quick note or reminder to the project notes file.
argument-hint: "<note text>"
```
- Appends note with timestamp to `.manuscript/NOTES.md`
- Format: `- [2026-04-07 14:30] Note text here`
- If NOTES.md doesn't exist, create it with a header
- Confirms: "Note added."

**`commands/scr/check-notes.md` (RUNTIME-06):**
```yaml
description: Review all project notes.
argument-hint: "[--clear]"
```
- Displays all notes from `.manuscript/NOTES.md`
- With `--clear`: archive notes to `.manuscript/archive/notes-{date}.md` and start fresh

**`commands/scr/plant-seed.md` (RUNTIME-06):**
```yaml
description: Plant a creative seed -- an idea, image, or fragment for future use.
argument-hint: "<idea>"
```
- Appends to `.manuscript/SEEDS.md` with timestamp and category tag (auto-detected: character, scene, dialogue, theme, world, other)
- Seeds can be referenced during discuss phase: "Check SEEDS.md for planted ideas"

**`commands/scr/troubleshoot.md` (RUNTIME-06):**
```yaml
description: Diagnose why something isn't working and suggest fixes.
argument-hint: "[description of problem]"
```
- Reads STATE.md, config.json, recent git log, and CONSTRAINTS.json
- Identifies common issues: wrong work type, missing prerequisites for a command, stuck state, missing files
- Suggests specific fix commands

**`commands/scr/thread.md` (RUNTIME-06):**
```yaml
description: Start or continue a focused conversation thread on a specific topic.
argument-hint: "<topic name>"
```
- Creates/continues `.manuscript/threads/{topic-slug}.md`
- Thread captures ongoing conversation about a specific aspect (e.g., "magic system", "villain motivation")
- Each entry timestamped with writer's thoughts and AI responses
- Useful for deep dives that don't fit discuss phase
