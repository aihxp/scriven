    Create commands/scr/cover-art.md following the established command pattern.

    Usage: /scr:cover-art [--kdp <trim_size>] [--series] [--prompt-only] [--element front|spine|back|full-wrap]

    The command generates cover art prompts saved to .manuscript/illustrations/cover/. Per product plan section 8.1:

    **Front cover prompt:** title, author name, tagline placement, primary imagery, typography style, mood
    **Spine prompt:** title, author, publisher logo area, width calculated from page count + paper type
    **Back cover prompt:** blurb text area, barcode placement zone (bottom-right, 2"x1.25"), author photo area, endorsement quotes area
    **Full wrap prompt:** combined front + spine + back as single image with KDP/IngramSpark dimensional specs

    Per D-02, genre-specific conventions:
    - Romance: couple/warm tones, flowing typography
    - Thriller/Mystery: dark palette, high contrast, bold sans-serif
    - Fantasy: sweeping landscape/scene, ornate typography
    - Sci-fi: space/technology imagery, clean modern type
    - Literary fiction: minimalist/typographic, muted palette
    - Children's: bright colors, character-focused, playful fonts
    - Horror: dark, atmospheric, distressed typography
    - Default/other: derive from WORK.md tone and genre

    Per ILL-02, KDP trim size specs with calculated spine width:
    - --kdp flag accepts trim size (e.g., "6x9", "5.5x8.5", "5x8")
    - Spine width = (page_count * paper_factor) + cover_thickness
    - Paper factors: white=0.002252in, cream=0.0025in, color=0.0032in (reuse Phase 5 constants)
    - Cover thickness: 0.06in
    - Full wrap width = front_width + spine_width + back_width + 0.125in bleed each side
    - Full wrap height = trim_height + 0.125in bleed top and bottom
    - Include DPI requirement: 300 DPI minimum for print

    Per D-03, --series flag:
    - When used, load ART-DIRECTION.md for visual consistency
    - Include series visual identity constraints (consistent typography, color palette, layout structure)
    - Reference prior cover designs if mentioned in ART-DIRECTION.md

    Per D-01, each prompt section uses structured format: Subject, Composition, Style, Color Palette, Mood, Technical Specs (dimensions, DPI, bleed).

    Load context: .manuscript/config.json (work_type, genre, title, author, page_count, paper_type), WORK.md (premise, tone), CHARACTERS.md (for character imagery), ART-DIRECTION.md (if exists, for style consistency). Check CONSTRAINTS.json for availability.

    Output files:
    - .manuscript/illustrations/cover/front-cover-prompt.md
    - .manuscript/illustrations/cover/spine-prompt.md
    - .manuscript/illustrations/cover/back-cover-prompt.md
    - .manuscript/illustrations/cover/full-wrap-prompt.md
    - Or single combined file when --element not specified
  </action>
