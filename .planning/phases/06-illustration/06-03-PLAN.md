    **illustrate-scene.md:**
    Usage: /scr:illustrate-scene <scene-ref> [--style <style>]

    Generate a scene-specific illustration prompt. Load:
    - .manuscript/config.json for work_type
    - data/CONSTRAINTS.json for availability and prerequisites (requires ART-DIRECTION.md + scene_drafted)
    - ART-DIRECTION.md from .manuscript/illustrations/ for visual style consistency
    - The drafted scene file referenced by <scene-ref> (use OUTLINE.md to resolve ref to file path)
    - CHARACTERS.md (or FIGURES.md for sacred) for character visual descriptions

    If ART-DIRECTION.md missing: "Run `/scr:art-direction` first to establish your visual style guide."
    If scene not drafted: "Scene '{ref}' has no draft yet. Draft it first with `/scr:draft {ref}`."

    Analyze the scene to extract:
    - Characters present (cross-reference with CHARACTERS.md for physical descriptions)
    - Setting/location (time of day, weather, indoor/outdoor, key objects)
    - Key action or emotional moment (the "illustratable moment" -- the most visually compelling beat)
    - Mood/atmosphere (tension, joy, mystery, etc.)

    Generate prompt per D-01 with sections:
    - Subject: The key moment described with all characters and their actions/positions
    - Composition: Suggested framing (wide establishing shot, close-up, medium shot), focal point, depth
    - Style: From ART-DIRECTION.md visual style
    - Color Palette: From ART-DIRECTION.md, adjusted for scene mood (darker for tense, warmer for intimate)
    - Mood: Derived from scene analysis
    - Characters: Visual descriptions of each character present (from CHARACTERS.md + ART-DIRECTION.md character specs)
    - Setting Details: Location-specific visual elements
    - Technical Specs: Recommended dimensions (full page: match trim size at 300 DPI, half page: half height)

    Save to .manuscript/illustrations/scenes/{scene-ref}-illustration-prompt.md

    **storyboard.md:**
    Usage: /scr:storyboard [--scene <ref>] [--act <number>]

    Generate storyboard frames for script work types. Load:
    - .manuscript/config.json for work_type
    - data/CONSTRAINTS.json for availability (script and visual groups only)
    - ART-DIRECTION.md if exists
    - OUTLINE.md for scene/beat structure
    - Drafted scene(s) for content

    If no --scene and no --act, generate storyboard for entire work (thumbnail overview).
    If --scene specified, generate detailed frames for that scene only.
    If --act specified, generate frames for all scenes in that act.

    Per D-05, each storyboard frame includes:
    - Frame number and scene/beat reference
    - Shot type: establishing, wide, medium, close-up, extreme close-up, over-the-shoulder, POV, aerial
    - Camera movement: static, pan (L/R/U/D), tilt, dolly, zoom, crane, handheld
    - Transition: cut, dissolve, fade, wipe, match cut
    - Description: What's in frame (characters, action, key props)
    - Dialogue/VO: Key line of dialogue or voiceover for this frame
    - Duration note: Beat length (quick, standard, lingering)
    - Mood/Lighting: Atmosphere for this frame

    Output as a sequence of frames with ASCII composition sketches:
    ```
    [FRAME 01] Scene 1, Beat 1 — ESTABLISHING
    Shot: Wide establishing | Camera: Static | Transition: FADE IN
    ┌─────────────────────────┐
    │  [SKY/HORIZON]          │
    │                         │
    │    [BUILDING]   [CHAR]  │
    │  [FOREGROUND ELEMENT]   │
    └─────────────────────────┘
    Description: ...
    Dialogue: ...
    ```

    Save to .manuscript/illustrations/storyboards/storyboard-{ref}.md

    **spread-layout.md:**
    Usage: /scr:spread-layout <spread-number> [--text-ratio <percent>]

    Generate children's book / picture book page spread layouts. Load:
    - .manuscript/config.json for work_type (must be in visual group: childrens_book, picture_book)
    - data/CONSTRAINTS.json for availability (visual group only)
    - ART-DIRECTION.md if exists
    - OUTLINE.md to get spread content (for picture books, each spread is an atomic unit)
    - Drafted spread content if exists

    Per D-04, output uses ASCII grid showing the two-page spread with labeled zones:

    ```
    SPREAD 3 — "The garden awakens"
    ┌──────────────────┬──────────────────┐
    │                  │                  │
    │  [ILLUSTRATION]  │  [ILLUSTRATION]  │
    │                  │                  │
    │                  ├──────────────────┤
    │                  │                  │
    │                  │  [TEXT]          │
    │                  │  "She stepped    │
    │                  │   into the       │
    │                  │   garden..."     │
    │  [BLEED]         │                  │
    └──────────────────┴──────────────────┘
    Left page: Full bleed illustration
    Right page: 60% illustration (top), 40% text (bottom)
    ```

    Include:
    - Text placement zone with actual text excerpt
    - Illustration zone with scene description prompt
    - Bleed areas marked
    - Gutter/binding area noted (center margin wider)
    - Text-to-illustration ratio (default 30/70 for picture books, configurable via --text-ratio)
    - Page dimensions (standard picture book: 10"x8" landscape spread = 20"x8")

    Save to .manuscript/illustrations/spreads/spread-{number}-layout.md

    **panel-layout.md:**
    Usage: /scr:panel-layout <page-number> [--panels <count>] [--style <style>]

    Generate comic panel layouts with composition notes. Load:
    - .manuscript/config.json for work_type (must be comic or graphic_novel per constraint comic_only)
    - data/CONSTRAINTS.json for availability and comic_only constraint
    - ART-DIRECTION.md if exists
    - OUTLINE.md for page content
    - Drafted page content if exists

    Generate panel layout with:
    - Panel grid: ASCII representation of panel arrangement on page
    - Panel count: 3-9 panels per page (default 6, configurable via --panels)
    - Panel sizes: varied (splash, half-page, quarter, strip, inset)
    - Gutter width: standard 0.125" between panels
    - For each panel:
      - Panel number
      - Content description (characters, action, dialogue placement)
      - Composition note (angle, depth, focus)
      - Balloon/caption placement zones

    Style options: traditional-grid, dynamic, manga, european, splash-heavy
    Default: traditional-grid

    ASCII panel layout example:
    ```
    PAGE 5
    ┌───────────────────────────┐
    │         PANEL 1           │  Establishing shot - wide
    │       (splash half)       │  [Character enters room]
    ├─────────────┬─────────────┤
    │   PANEL 2   │   PANEL 3   │  P2: Close-up face
    │             │             │  P3: Hand on doorknob
    ├─────────────┼─────────────┤
    │   PANEL 4   │   PANEL 5   │  P4: Reaction shot
    │             │             │  P5: Wide - room revealed
    └─────────────┴─────────────┘
    ```

    Save to .manuscript/illustrations/panels/page-{number}-layout.md
