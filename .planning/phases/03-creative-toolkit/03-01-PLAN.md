Create 3 command files following the exact markdown skill file pattern from new-character.md:

**character-sheet.md** (CHAR-02):
- YAML frontmatter: `description: Display or edit a specific character's full profile.` and `argument-hint: "[name] [--edit]"`
- Title: `/scr:character-sheet -- View or Edit Character Profile`
- Instruction loads: config.json (work_type), CONSTRAINTS.json (file_adaptations), the adapted characters file (CHARACTERS.md / FIGURES.md / CONCEPTS.md), STYLE-GUIDE.md
- DISPLAY MODE (default): Find character by name (case-insensitive), present full profile highlighting voice anchor attributes per D-01: speech patterns, vocabulary register, sentence length, verbal tics, internal monologue style (5-8 concrete attributes)
- EDIT MODE (--edit): Walk through sections interactively, update characters file, commit
- Work-type adaptation: Read CONSTRAINTS.json file_adaptations to determine file name and terminology (character/figure/concept)
- Edge cases: Character not found, multiple partial matches, empty characters file
- Commit message: `character: update {name} profile`

**character-voice-sample.md** (CHAR-04):
- YAML frontmatter: `description: Generate a dialogue sample to preview a character's voice before drafting.` and `argument-hint: "[name]"`
- Title: `/scr:character-voice-sample -- Preview Character Voice`
- Instruction loads: config.json, CONSTRAINTS.json (file_adaptations), adapted characters file, STYLE-GUIDE.md
- Generate exactly 5 lines of dialogue for the named character, showcasing their speech patterns, vocabulary register, verbal tics, and emotional state
- Include brief stage direction / action beats between lines
- Show the sample and ask writer to approve, adjust, or regenerate
- Prerequisites: CHARACTERS.md + STYLE-GUIDE.md must exist (from CONSTRAINTS.json feature_prerequisites)
- Work-type adaptation: sacred uses "figure voice sample", academic uses "concept voice"

**cast-list.md** (CHAR-06):
- YAML frontmatter: `description: Display the roster of all characters with roles and brief descriptions.`
- Title: `/scr:cast-list -- Character Roster`
- Instruction loads: config.json, CONSTRAINTS.json (file_adaptations), adapted characters file
- Display formatted table: Name, Role, Arc Type, Status (active/deceased/mentioned), 1-line summary
- Group by role category (protagonist, antagonist, supporting, mentioned)
- Work-type adaptation: sacred uses "figures list", academic uses "concepts list"

Also verify CHAR-01 (existing new-character.md): Read the existing file and confirm it includes voice anchor sections covering D-01's 5-8 attributes (speech patterns, vocabulary register, sentence length, verbal tics, internal monologue style). The existing command already has voice and speech_patterns interview sections which satisfy this. No changes needed unless attributes are missing.
