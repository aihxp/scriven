Create 6 structure management command files. ALL share the draft-safe pattern from D-07 and work-type polymorphism from CONSTRAINTS.json hierarchy.

**Common preamble for all 6 commands** (include in each Instruction section):
1. Load `.manuscript/config.json` to get `work_type`
2. Load `data/CONSTRAINTS.json` and find `work_types[work_type].hierarchy` to determine unit terminology
3. Load `OUTLINE.md` and `STATE.md`
4. Before any modification: scan `.manuscript/drafts/` for files matching affected units
5. If drafted content is affected: list all affected files with content summary, warn the writer, require explicit confirmation before proceeding (D-07: NEVER silently move or delete drafted prose)
6. After modification: update OUTLINE.md, STATE.md, and PLOT-GRAPH.md if arc positions changed
7. Commit with message: `structure: {operation} {unit_type} {details}`

**add-unit.md** (part of STRUCT-06):
- YAML frontmatter: `description: Add a new unit to the end of the outline.` and `argument-hint: "[title]"`
- Title: `/scr:add-unit -- Add New Unit`
- Add a new unit (chapter/scene/section/etc. per hierarchy) to the end of OUTLINE.md
- Prompt for: title, brief summary, placement in arc (if PLOT-GRAPH.md exists)
- Update numbering in OUTLINE.md
- Adapted names: novel uses "add-chapter", screenplay uses "add-scene", etc.

**insert-unit.md** (part of STRUCT-06):
- YAML frontmatter: `description: Insert a new unit at a specific position in the outline.` and `argument-hint: "[position] [title]"`
- Title: `/scr:insert-unit -- Insert Unit at Position`
- Insert at specified position, renumber subsequent units
- Check if insertion affects existing draft file numbering, warn if so
- Adapted names per work type

**remove-unit.md** (part of STRUCT-06):
- YAML frontmatter: `description: Remove a unit from the outline with draft safety checks.` and `argument-hint: "[unit-id]"`
- Title: `/scr:remove-unit -- Remove Unit`
- CRITICAL: If the unit has a draft file, show its word count and first 3 lines, explicitly warn, and require "yes" confirmation
- Offer to archive (move to `.manuscript/archive/`) rather than delete
- Renumber remaining units, update all references

**split-unit.md** (part of STRUCT-06):
- YAML frontmatter: `description: Split one unit into two at a specified point.` and `argument-hint: "[unit-id] [split-point]"`
- Title: `/scr:split-unit -- Split Unit`
- If drafted: ask where to split the content (by paragraph, scene break, or word count)
- Create two new entries in OUTLINE.md, prompt for new titles
- Move/split draft content accordingly with confirmation

**merge-units.md** (part of STRUCT-06):
- YAML frontmatter: `description: Merge two adjacent units into one.` and `argument-hint: "[unit-id-1] [unit-id-2]"`
- Title: `/scr:merge-units -- Merge Adjacent Units`
- Only allow merging adjacent units (show error for non-adjacent)
- If both have drafts: concatenate with a scene break marker between them
- Prompt for merged unit title, update OUTLINE.md numbering

**reorder-units.md** (part of STRUCT-06):
- YAML frontmatter: `description: Reorder units in the outline by moving a unit to a new position.` and `argument-hint: "[unit-id] [new-position]"`
- Title: `/scr:reorder-units -- Reorder Outline Units`
- Move unit from current position to new position, renumber all affected units
- If drafted: rename draft files to match new numbering
- Update PLOT-GRAPH.md arc positions if they reference unit numbers
- Show before/after outline comparison for confirmation
