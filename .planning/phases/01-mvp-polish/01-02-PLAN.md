    **bin/install.js:** Add module.exports at the bottom of the file, AFTER the `main().catch(...)` call. The exports must be conditional so that requiring the file for testing does NOT trigger the interactive installer. Change the bottom of the file from:

    ```javascript
    main().catch((err) => {
      console.error(c('red', '\nInstallation failed:'), err.message);
      process.exit(1);
    });
    ```

    To:

    ```javascript
    // Only run interactive installer when executed directly
    if (require.main === module) {
      main().catch((err) => {
        console.error(c('red', '\nInstallation failed:'), err.message);
        process.exit(1);
      });
    }

    module.exports = { copyDir, RUNTIMES };
    ```

    This means moving the `main().catch()` inside the `require.main === module` guard. The `copyDir` and `RUNTIMES` exports let tests validate file copying and runtime configurations without triggering the interactive CLI.
