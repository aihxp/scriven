Create `docs/architecture.md` -- the architecture overview per D-03 (skill system design, CONSTRAINTS.json schema, file structure, agent orchestration, fresh-context-per-unit) and D-01 (friendly tone).

Structure:
1. **Overview** -- Scriven is a pure skill system. No compiled code, no runtime dependencies. AI agents read markdown command files and execute them. The system runs inside the agent's existing environment.
2. **Skill System Design** -- How it works: commands are markdown files with YAML frontmatter. The AI agent reads the command file, follows its instructions, and uses its built-in tools (Read, Write, Bash). No SDK, no API, no framework.
3. **CONSTRAINTS.json Schema** -- The central registry. Document each top-level key:
   - `work_type_groups` -- how work types are categorized (prose, script, academic, visual, poetry, interactive, speech_song, sacred)
   - `work_types` -- individual type definitions with `label`, `group`, `hierarchy`, `unit_names`, `voice_register`, `available_commands`
   - `commands` -- command registry with `category`, `available_for`, `description`
   - Show a concrete work_type entry (e.g., `novel`) and a command entry as examples
4. **File Structure** -- Map the codebase:
   - `commands/scr/` -- 89 command markdown files (one per `/scr:*` command)
   - `agents/` -- 6 specialized agents (drafter, voice-checker, continuity-checker, plan-checker, researcher, translator)
   - `data/CONSTRAINTS.json` -- central constraint registry
   - `data/demo/` -- pre-baked demo project
   - `data/export-templates/` -- Typst, CSS, LaTeX, DOCX templates
   - `templates/` -- 9 project templates + sacred variants
   - `bin/install.js` -- multi-platform installer
   - `docs/` -- documentation suite
   - `.manuscript/` -- per-project working directory (created by commands, not shipped)
5. **Agent Orchestration** -- How commands invoke agents:
   - Commands like `/scr:draft` invoke the drafter agent in fresh context
   - Commands like `/scr:autopilot` orchestrate multiple agent invocations
   - Each agent receives specific files (STYLE-GUIDE.md always first)
   - Agents are stateless -- they receive everything they need per invocation
6. **Fresh Context per Unit** -- The core architectural pattern:
   - WHY: prevents voice drift, context bloat, keeps each unit focused
   - HOW: drafter agent invoked once per atomic unit (scene, subsection, passage, stanza) with clean context
   - WHAT it receives: STYLE-GUIDE.md, unit PLAN, relevant CHARACTERS, previous unit tail (200 words), THEMES excerpt, WORK excerpt
   - WHAT it does NOT receive: full manuscript, other unit drafts, revision history
   - This is the key to voice fidelity -- the core value of Scriven
7. **Installer Architecture** -- How `bin/install.js` works:
   - Platform detection (Claude Code, Cursor, Windsurf, Gemini CLI, etc.)
   - Two installation strategies: command-directory (copies command files) vs skill-file (generates SKILL.md manifest)
   - Runtime type classification: 'commands' vs 'skills'
8. **Voice DNA Pipeline** -- How STYLE-GUIDE.md flows through the system:
   - Created by `/scr:new-work` from voice sample
   - Loaded into every drafter invocation
   - Checked by voice-checker agent
   - 15+ dimensions profiled
9. **Design Principles** -- Zero dependencies, plan is canonical (section 15 in product plan), backward compatibility, progressive disclosure

Use ASCII art for the agent orchestration flow (e.g., showing how autopilot chains draft -> voice-check -> continuity-check). Keep the tone practical -- a developer should finish reading and think "I get how this works" not "I need to read more docs."
