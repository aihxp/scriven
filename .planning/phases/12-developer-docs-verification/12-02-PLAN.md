  <action>
Systematic verification pass per D-04. Execute these checks in order:

**Step 1: Build canonical lists from the codebase**
- List all `commands/scr/*.md` files -> extract command names (strip .md extension)
- List all `agents/*.md` files -> extract agent names
- List all `templates/*` files -> extract template names
- List all `data/export-templates/*` files -> extract template names

**Step 2: Extract all references from documentation**
For each file in `docs/*.md` and `README.md`:
- Extract all `/scr:{name}` command references using grep
- Extract all file path references (e.g., `commands/scr/`, `agents/`, `templates/`, `data/`, `.manuscript/`, `bin/`)
- Extract all agent name references (drafter, voice-checker, etc.)
- Extract all work type name references and cross-check against CONSTRAINTS.json `work_types` keys

**Step 3: Cross-reference and identify discrepancies**
For each extracted reference:
- Check if the referenced command exists in the canonical command list
- Check if the referenced file path exists on disk
- Check if the referenced agent exists in the canonical agent list
- Log all mismatches

**Step 4: Fix all discrepancies**
For each mismatch found:
- If command name is wrong (typo or renamed), fix to correct name
- If file path is wrong, fix to correct path
- If reference is to something that was never created, remove the reference or note it
- If a command count is stated (e.g., "96+ commands"), update to actual count (89 commands in commands/scr/)

**Step 5: Verify cross-links between docs**
- Check that links between docs (e.g., "see [Voice DNA Guide](voice-dna.md)") point to files that exist
- Check that README.md links to all docs that exist in docs/
- Verify README.md includes links to the new contributing.md and architecture.md from Plan 01

**Step 6: Update README.md**
- Add links to `docs/contributing.md` and `docs/architecture.md` in the appropriate section
- Verify all existing doc links still work

**Step 7: Write verification report**
After all fixes, report:
- Total references checked
- References that were correct
- References that were fixed (list each with before/after)
- Any references that could not be resolved

Output the verification report as the final section of the SUMMARY.md file (not a separate file).
  </action>
