  <action>
Create the `data/export-templates/` directory and three template files.

**scriven-book.typst** -- Typst book interior template for Pandoc `--template` flag:
- Accept page dimensions as Pandoc variables: paperwidth (default 6in), paperheight (default 9in), margin-inside (0.75in), margin-outside (0.5in), margin-top (0.75in), margin-bottom (0.75in)
- Page numbering: centered bottom, starting after front matter
- Running headers: book title left, chapter title right (recto/verso mirrored)
- Font: Libertinus Serif 11pt (or fallback to serif), 1.4 line spacing
- Chapter headings: new page, centered, large font, extra spacing
- Do NOT hardcode LTR assumptions -- use parameterized text direction for Phase 7 RTL compatibility
- Widow/orphan control: min 2 lines

**scriven-epub.css** -- EPUB stylesheet for Pandoc `--css` flag:
- Body: Georgia/serif fallback, 1.5 line-height, 1em paragraph indent (not first paragraph)
- Headings: sans-serif, graduated sizes using em units (NOT px -- KDP requirement)
- Chapter breaks: page-break-before on h1
- Images: max-width 100%, centered
- Blockquotes: indented, italic
- No fixed widths or absolute positioning (EPUB reflow requirement)
- Alt text reminder comment for accessibility

**scriven-academic.latex** -- LaTeX template for Pandoc `--template` flag:
- documentclass: article (default) with options for report/book
- 12pt font, 1-inch margins via geometry package
- Bibliography support via biblatex/biber
- Abstract environment
- Section numbering, table of contents
- Author, title, date from Pandoc variables
- Citation support with citeproc compatibility

Create the main export command file per D-01 (AI agent instructions that invoke pandoc/typst via shell, not compiled code).

**YAML frontmatter:**
- description: "Compile and export manuscript to publication-ready formats."
- argument-hint: "--format <format> [--formatted] [--print-ready]"

**Structure** (single command file, format dispatching per research Pattern 3):

**STEP 1: LOAD CONTEXT**
- Load config.json for work type, title, author
- Load CONSTRAINTS.json -- check exports section for format availability by work type group
- If requested format not available for current work type, tell writer and list available formats

**STEP 2: CHECK PREREQUISITES** (per D-02)
- For markdown: no external tools needed
- For docx/epub/latex: `command -v pandoc` -- if missing, provide platform-specific install instructions (brew/apt/choco) and stop
- For pdf: also check `command -v typst` -- if missing, provide install instructions and stop
- For fdx: check `command -v screenplain` (covered in Plan 02)
- For fountain: no external tools needed (agent writes Fountain directly)

**STEP 3: ASSEMBLE MANUSCRIPT** (per D-03)
- Read .manuscript/OUTLINE.md -- parse scene/chapter list for document order
- Scan .manuscript/front-matter/ -- sort by numeric prefix (01-half-title.md, 02-series-title.md, etc.)
- Read body drafts from .manuscript/drafts/body/ -- order by OUTLINE.md scene list
- Scan .manuscript/back-matter/ -- sort alphabetically
- Concatenate with `\newpage` page break markers between sections
- Warn about missing units (listed in OUTLINE but no draft file), skip them
- Warn if no front matter exists -- suggest `/scr:front-matter`
- Write assembled file to .manuscript/output/assembled-manuscript.md
- Generate metadata.yaml from config.json (title, author, language, rights, date) and write to .manuscript/output/metadata.yaml

**STEP 4: EXPORT BY FORMAT**
Use exact shell invocations from research:

- **markdown** (EXP-01): The assembled file IS the output. Copy to .manuscript/output/manuscript.md
- **docx** (EXP-02): `pandoc .manuscript/output/assembled-manuscript.md -o .manuscript/output/manuscript.docx --reference-doc=data/export-templates/scriven-manuscript.docx --metadata-file=.manuscript/output/metadata.yaml --toc --toc-depth=1`
- **docx --formatted** (EXP-03): Same but `--reference-doc=data/export-templates/scriven-formatted.docx` and `--toc-depth=2`, output as manuscript-formatted.docx
- **pdf** (EXP-04): `pandoc ... -o .manuscript/output/manuscript.pdf --pdf-engine=typst --metadata-file=... --toc --toc-depth=1`
- **pdf --print-ready** (EXP-05): `pandoc ... -o .manuscript/output/manuscript-print.pdf --pdf-engine=typst --template=data/export-templates/scriven-book.typst --metadata-file=... --toc --toc-depth=2 -V paperwidth=6in -V paperheight=9in -V margin-inside=0.75in -V margin-outside=0.5in -V margin-top=0.75in -V margin-bottom=0.75in` (trim size from config.json if set, else defaults)
- **epub** (EXP-06): `pandoc ... -o .manuscript/output/manuscript.epub --metadata-file=... --epub-cover-image=.manuscript/output/cover.jpg --css=data/export-templates/scriven-epub.css --toc --toc-depth=2 --split-level=1` (skip --epub-cover-image if no cover exists, warn)

**STEP 5: REPORT**
- Show output file path and size
- For EPUB: suggest running EPUBCheck if java is available (`java -jar epubcheck.jar`)
- Note any warnings from assembly step

NOTE: Secondary formats (fountain, fdx, latex) and packages (kdp, ingram, query, submission) will be added in Plan 02. Leave clear section markers for extension.
